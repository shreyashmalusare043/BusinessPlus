═══════════════════════════════════════════════════════════════════════════════
  AUTOMATIC STATUS TRANSITIONS
═══════════════════════════════════════════════════════════════════════════════

Pending → In Progress:
  Trigger: First work log entry (quantity_completed > 0)
  Condition: Current status is 'pending'
  Action: Status changes to 'in_progress'

In Progress → Completed:
  Trigger: Total completed reaches or exceeds ordered quantity
  Condition: quantity_completed >= quantity_ordered
  Actions:
    • Status changes to 'completed'
    • Completion date set to current date
    • Work order removed from Work Track dropdown


═══════════════════════════════════════════════════════════════════════════════
  DATABASE AUTOMATION
═══════════════════════════════════════════════════════════════════════════════

✅ Trigger Function: update_work_order_status()
   • Fires after INSERT or UPDATE on work_logs
   • Only when work_order_id is linked
   • Calculates total completed quantity
   • Updates status based on progress
   • Sets completion date when finished

✅ Calculation Function: calculate_work_order_completed_quantity()
   • Sums all quantity_completed from work logs
   • Returns total for a work order
   • Used by trigger and view

✅ Progress View: work_orders_with_progress
   • Joins work_orders with work_logs
   • Calculates quantity_completed (SUM)
   • Calculates quantity_pending (ordered - completed)
   • Always up-to-date
   • Used by all queries
