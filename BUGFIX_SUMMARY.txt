═══════════════════════════════════════════════════════════════════════════════
  ISSUE #1: Work Order List Showing Incorrect Quantities
═══════════════════════════════════════════════════════════════════════════════

PROBLEM REPORTED:
─────────────────────────────────────────────────────────────────────────────
• Work Order List showing:
  - Qty Completed: 0
  - Pending: 500
  - Status: COMPLETED
• But when clicking to view details, everything shows correctly
• Inconsistency between list view and detail view

ROOT CAUSE:
─────────────────────────────────────────────────────────────────────────────
• getWorkOrders() function was querying from 'work_orders' table directly
• The base 'work_orders' table does NOT have quantity_completed or 
  quantity_pending fields
• These fields are calculated in the 'work_orders_with_progress' VIEW
• Detail page was using the view correctly, but list page was not

THE FIX:
─────────────────────────────────────────────────────────────────────────────
Updated three API functions to use 'work_orders_with_progress' view:

1. getWorkOrders()
   Before: .from('work_orders')
   After:  .from('work_orders_with_progress')

2. getWorkOrdersByStatus()
   Before: .from('work_orders')
   After:  .from('work_orders_with_progress')

3. getWorkOrdersByCustomer()
   Before: .from('work_orders')
   After:  .from('work_orders_with_progress')

RESULT:
─────────────────────────────────────────────────────────────────────────────
✓ Work Order List now shows correct Qty Completed (green color)
✓ Work Order List now shows correct Pending quantity (primary color)
✓ Status reflects actual completion state
✓ Consistent data across all pages (list, detail, dashboard)
✓ Real-time updates reflected immediately

EXAMPLE:
─────────────────────────────────────────────────────────────────────────────
Work Order: 500 pieces ordered, 500 completed

Before Fix:
  List View:  Qty Completed: 0, Pending: 500, Status: COMPLETED ❌
  Detail View: Qty Completed: 500, Pending: 0, Status: COMPLETED ✓

After Fix:
  List View:  Qty Completed: 500, Pending: 0, Status: COMPLETED ✓
  Detail View: Qty Completed: 500, Pending: 0, Status: COMPLETED ✓


═══════════════════════════════════════════════════════════════════════════════
  ISSUE #2: Work Tracking Dialog Not Scrollable
═══════════════════════════════════════════════════════════════════════════════

PROBLEM REPORTED:
─────────────────────────────────────────────────────────────────────────────
• Work Tracking form dialog gets stuck
• Cannot scroll down to reach notes field
• Form content is longer than screen height
• Users unable to complete form entry

ROOT CAUSE:
─────────────────────────────────────────────────────────────────────────────
• DialogContent had no max-height constraint
• No overflow-y-auto for vertical scrolling
• Long form with work order details card exceeded viewport height
• When work order is selected, details card adds significant height
• Fixed dialog height without scroll capability

THE FIX:
─────────────────────────────────────────────────────────────────────────────
Updated DialogContent className:

Before: className="max-w-2xl"
After:  className="max-w-2xl max-h-[90vh] overflow-y-auto"

Changes:
• max-h-[90vh]: Limits dialog height to 90% of viewport height
• overflow-y-auto: Enables vertical scrolling when content exceeds height

RESULT:
─────────────────────────────────────────────────────────────────────────────
✓ Dialog scrolls smoothly on all screen sizes
✓ All form fields accessible via scroll
✓ Notes field reachable at bottom of form
✓ Work order details card visible without blocking form
✓ Better UX on smaller screens (laptops, tablets)
✓ No content cut off or hidden

FORM STRUCTURE:
─────────────────────────────────────────────────────────────────────────────
Work Tracking Dialog Content (top to bottom):

1. Dialog Header (fixed at top)
2. Date field
3. Work Type dropdown
4. Work Order selection dropdown
5. Work Order Details Card (when order selected) ← Adds height
6. Company Name field
7. Quantity Completed field (when order selected)
8. Machine Hours field
9. Jobs Completed field
10. Notes field ← Previously unreachable
11. Cancel / Create buttons (at bottom)

Now all fields are accessible via smooth scrolling!
